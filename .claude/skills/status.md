# /status - 블로그 빌드 진행 상황 확인

## Triggers
- 세션 시작 시 "어디까지 했지", "현재 상태", "다음 뭐 해야 해" 등의 질문
- PLAN.md 언급 시
- 작업 시작 전 컨텍스트 파악이 필요할 때

## Behavioral Flow
1. PLAN.md 읽기 → 체크된 항목 / 미완료 항목 분류
2. `src/` 디렉토리 실제 구조 확인
3. PLAN.md 상태와 실제 구현 상태 비교 (불일치 감지)
4. 다음 진행할 작업 명확하게 제안

## Output Format
```
## 진행 현황
✅ 완료: [항목들]
⏳ 미완료: [항목들]

## 실제 구현 상태
[src/ 구조 기반 확인 결과]

## 불일치 항목
[PLAN.md와 실제 구현이 다른 부분]

## 다음 작업
[구체적인 다음 단계]
```
